## Feature Output

The entire `src/` directory is a **generated artefact** — it is git-ignored and regenerated by `scripts/sync-src.py`.

`src/` is fully auto-generated and git-ignored.

This is the canonical source for the install script's entry point, and is copied into each feature's `src/*/install.sh` by `scripts/sync-src.py`.


- `src/`: **Fully auto-generated and gitignored** — assembled from `features/` by `scripts/sync-src.py`. Consumed by the devcontainer CLI and build tooling. **Never edit files here directly.**
  - `src/*/`: Per-feature generated directory.
    - `src/*/devcontainer-feature.json`: Generated from `features/*/metadata.yaml` by `scripts/sync-src.py`.
    - `src/*/install.sh`: Generated copy of `features/install.sh` (POSIX shim). Overwritten on every sync.
    - `src/*/install.bash`: Generated assembled installer — autogenerated header + body from `features/*/install.bash`. Overwritten on every sync.
    - `src/*/_lib/`: Generated copy of `lib/`. Overwritten on every sync.
    - `src/*/dependencies/`: Generated from the `_dependencies:` key in `features/*/metadata.yaml` by `scripts/sync-src.py`.
    - `src/*/files/`: Copied from `features/*/files/` by `scripts/sync-src.py` (only for features that have a `files/` directory).


```
src/<feature>/
├── devcontainer-feature.json   ← Generated from metadata.yaml (git-ignored)
├── install.sh                  ← Generated bootstrap/POSIX shim (git-ignored)
├── install.bash                ← Generated: header + body from features/ (git-ignored)
├── _lib/                       ← Generated copy of lib/ (git-ignored)
├── dependencies/               ← Generated from metadata.yaml _dependencies: key
│   └── base.yaml               ← OS packages required before install.bash runs
└── files/                      ← Copied from features/<feature>/files/ (git-ignored)
```

The devcontainer CLI calls `install.sh` (the bootstrap) at image build time.
The bootstrap ensures `bash` is available, then execs
`install.bash "$@"`, which is the real installer written in bash ≥4.
`install.bash` sources helpers from `_lib/` — the local copy
of the shared library.



Reasons for not committing:
- They are exact copies — committing them would create merge conflicts and
  make diffs noisy every time a library file is touched.
- CI regenerates them from the canonical sources on every run, so the working
  tree is always correct.
- The git-ignore makes it immediately apparent when someone accidentally edits
  a generated copy (the change disappears on the next sync).
