# Install Script

`features/<feature-id>/install.bash` contains the **body only** — no shebang, no `set -euo pipefail`, no path setup. The autogenerated header prepended by `sync-src.py` provides all of that. The sections below describe the conventions for the body.


> **Check the library first.** Before writing inline logic, confirm there is no existing function: use `os__kernel`/`os__arch` instead of `uname -s`/`uname -m`; `os__font_dir` instead of manual platform detection; `github__latest_tag` instead of a hand-rolled API call; `checksum__verify_sidecar` instead of inline sha256sum logic; `users__is_root`/`users__default_prefix` instead of `id -u` checks; `users__resolve_list` + `users__set_login_shell` instead of manual deduplication and `chsh` calls.


## Library-First Principle

**Always check `lib/` before writing inline logic.** If a function already exists in the shared library, use it — do not reimplement it. Common mistakes to avoid:

- Calling `uname -s` / `uname -m` directly → use `os__kernel` / `os__arch` instead.
- Detecting the font directory with an if/elif block → use `os__font_dir`.
- Hand-rolling a GitHub API call with curl → use `github__fetch_release_json`, `github__latest_tag`, or `github__release_tags`.
- Implementing SHA-256 verification inline → use `checksum__verify_sha256` or `checksum__verify_sha256_sidecar`.
- Resolving devcontainer user lists with a local associative array → use `users__resolve_list`.
- Calling `chsh` manually for a list of users → use `users__set_login_shell`.

**When adding new logic**, ask: could this be useful in more than one feature, or does it encapsulate a detail that is easy to get wrong? If yes, add it to `lib/` rather than keeping it inline. After adding to `lib/`, run `python3 scripts/sync-src.py` to propagate it to all features.

### Logging setup

Always set up logging immediately after sourcing the library, with the EXIT trap to ensure cleanup runs even on error:

```bash
logging__setup
logging__entry "My Feature"
trap 'logging__cleanup' EXIT
```

`logging__setup` redirects stdout and stderr through `tee` into a temp file. `logging__cleanup` (called on EXIT) flushes the captured output to `$LOG_FILE` if set. Call `logging__set_level` after parsing options so `--log_level` takes effect.

### Dual-Mode Argument Parsing

The devcontainer CLI passes options as environment variables,
whereas direct script invocation uses CLI flags.
Every feature must work in two modes:

- **Devcontainer mode** (no CLI arguments): options injected as environment variables.
- **CLI mode** (arguments present): called as `install.bash --version 1.2.3 --log_level trace`.

```bash
if [ "$#" -gt 0 ]; then
  # Reset all variables before parsing — avoid inheriting caller's env.
  VERSION=""
  LOG_LEVEL=""
  LOG_FILE=""

  while [[ $# -gt 0 ]]; do
    case $1 in
      --version)   shift; VERSION="$1"; shift ;;
      --log_level) shift; LOG_LEVEL="$1"; shift ;;
      --log_file)  shift; LOG_FILE="$1"; shift ;;
      --help|-h)   __usage__; exit 0 ;;
      --*)         logging__error "Unknown option: '${1}'"; exit 1 ;;
      *)           logging__error "Unexpected argument: '${1}'"; exit 1 ;;
    esac
  done
else
  logging__info "Reading environment variables."
  [ "${VERSION+defined}" ] && logging__read "Argument 'version': '${VERSION}'"
fi

# Apply defaults AFTER parsing
[ -z "${OPTNAME-}" ] && OPTNAME="default_value"
[ -z "${LOG_LEVEL-}" ] && LOG_LEVEL=info
[ -z "${LOG_FILE-}" ] && LOG_FILE=""
logging__set_level
```


The `[ "${VAR+defined}" ]` pattern (without `-z`) checks whether the variable was **set**, even if set to an empty string — distinguishing "caller set it empty" from "caller did not set it at all".

Include a `__usage__()` function that prints the full option list to stderr and is invoked for `--help`.

Apply defaults after argument parsing using `:-`:

```bash
: "${VERSION:=latest}"
: "${LOG_LEVEL:=info}"
: "${LOG_FILE:=}"
logging__set_level   # call after defaults so log_level takes effect
```

Validate early and exit with a clear message:

```bash
os__require_root

if [[ ! "$VERSION" =~ ^[0-9]+\.[0-9]+\.[0-9]+$|^latest$ ]]; then
  logging__error "Invalid version: '${VERSION}'"
  exit 1
fi
```


### OS Package Dependencies

ospkg__run --manifest "${_BASE_DIR}/dependencies/base.yaml" --skip_installed

`--skip_installed` skips packages already present (idempotent). Omit `--skip_installed` when upgrading is desired.


## Main logic

Write idempotent logic wherever possible — check whether work has already been done before doing it:

```bash
if command -v mytool > /dev/null 2>&1; then
  logging__info "mytool already installed — skipping."
else
  logging__install "Installing mytool..."
  # ... installation steps ...
fi
```

When writing new logic, ask: could this be useful in more than one feature, or does it encapsulate a non-trivial detail easy to get wrong? If yes, add it to `lib/` and run `just sync-src` to propagate it.

For installer logic that is used both by DevFeats internals and by user-facing features, place it under `lib/install/` and expose exactly one public entrypoint (for example `install__oras`). Keep resolve/download/verify helpers private in that module. Feature `install.bash` files should be thin wrappers that pass user options with `context=user`, while internal callers pass `context=internal`. This keeps ownership, cleanup, and promotion semantics consistent across all call sites.

End the script with a success message:

```bash
logging__success "my-feature setup complete."
```

### OS package installation

Use `ospkg__run` for manifests and `ospkg__install` for individual packages:

- **`ospkg__run --manifest <file>`** — full pipeline: detect PM, update package lists, install packages from the manifest, run pre/post scripts, clean cache.
- **`ospkg__install <pkg>...`** — installs named packages directly (with idempotency check on APT/DNF).

When calling both in the same script, use `--keep_cache` on `ospkg__run` and call `ospkg__clean` once at the end:

```bash
ospkg__run --manifest "$_MANIFEST" --skip_installed --keep_cache
ospkg__install some-extra-package
ospkg__clean
logging__success "my-feature setup complete."
```

### Static files

Place static files under `features/<feature-id>/files/`. Reference them in `install.bash` via `_FILES_DIR`:

```bash
_FILES_DIR="${_BASE_DIR}/files"
cp "${_FILES_DIR}/my-config.conf" /etc/my-config.conf
```

During sync, `files/` is copied to `src/<feature-id>/files/` (not symlinked).
